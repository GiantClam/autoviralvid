# PPT 审核规范手册

> 适用场景：工作汇报 / 融资路演 / 课程讲义
> 规则权重说明：🔴 必须满足 | 🟡 关键项 | 🔵 加分项

---

## 一、工作汇报

### 1.1 结构逻辑

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 封面包含：项目名、汇报人、日期、一句核心结论 | 核心结论应在10字以内，直接说明最重要的发现或建议 |
| 🔴 必须 | 第2页为执行摘要，可独立阅读，无需翻后续页 | 包含：现状快照 / 核心问题 / 建议方案 / 所需决策，共4项 |
| 🔴 必须 | 每页标题是结论句，而非话题词 | 错误示例：「销售分析」→ 正确：「Q3销售同比增长23%，主要来自华北区」 |
| 🔴 必须 | 所有标题连读可构成完整叙事逻辑 | 把每页标题复制到一页，检查是否讲了一个完整故事 |
| 🟡 关键 | 行动项页明确：谁、做什么、何时完成 | 避免「加强推进」「持续优化」等无法考核的表述 |
| 🟡 关键 | 附录页备齐但不主动讲 | 详细数据、方法论、假设条件放附录，备问答时翻阅 |
| 🔵 加分 | 设置目录页并标注各部分预计时长 | 帮助听众预期节奏，也方便跳读 |

### 1.2 内容表达

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 每页只传递一个核心观点 | 如果一页有两个结论，拆成两页 |
| 🔴 必须 | 正文文字 ≤40字/页，标题 ≤20字 | 超出部分移入备注栏，演讲时口头补充 |
| 🟡 关键 | 所有数字做对比：vs 去年 / vs 目标 / vs 行业 | 孤立数字无意义，对比才能产生判断 |
| 🟡 关键 | 无模糊表述：「基本上」「相关情况」「进一步」等 | 每个描述应可量化或可验证 |
| 🟡 关键 | 因果关系明确，不只陈述现象 | 不只写「销售下滑」，要写「因为X导致销售下滑Y%」 |
| 🔵 加分 | 使用具体案例或真实客户反馈佐证 | 一个真实案例比三条理论分析更有说服力 |
| 🔵 加分 | 用主动语态，避免被动表述 | 「团队完成了X」优于「X工作已由团队完成」 |

### 1.3 数据规范

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 每张图表标注数据来源和时间范围 | 格式：来源：XX数据库，统计周期：2024.01-09 |
| 🔴 必须 | 同一汇报中相同指标口径统一 | 例如「用户数」需全程使用同一定义：注册用户/活跃用户/付费用户 |
| 🟡 关键 | 图表类型与数据类型匹配 | 趋势用折线图，比例用饼/环图，对比用条形图，分布用散点图 |
| 🟡 关键 | 坐标轴从0开始，不人为放大差异 | Y轴截断会误导判断，特殊情况须注明 |
| 🟡 关键 | 预测数据与实际数据视觉区分（虚线/不同色） | 避免将预测值与已知数据混在同一实线中 |
| 🔵 加分 | 关键数字使用大字号单独展示 | 用「数字大图」页面展示最重要的1个指标 |

### 1.4 视觉设计

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 全篇使用统一字体，正文 ≥18pt | 标题建议28-32pt，正文18-22pt，注释 ≤14pt |
| 🔴 必须 | 配色不超过3种，以企业色为主 | 主色+强调色+中性色，不使用荧光色、高饱和色 |
| 🟡 关键 | 元素对齐：文本框、图表、图片均与网格对齐 | 开启参考线检查，视觉对齐比内容对齐更影响专业感 |
| 🟡 关键 | 每页有足够留白，内容不超过页面60%面积 | 拥挤的页面让人感到焦虑，留白=重点突出 |
| 🟡 关键 | 图表简洁：删除多余网格线、图例、边框 | 默认图表往往有很多无意义装饰，逐项删除 |
| 🔵 加分 | 重要数字/关键词使用强调色高亮 | 每页只高亮1-2处，超过则失去强调效果 |
| 🔵 加分 | 图片使用高分辨率，无拉伸变形 | 图片宽高比锁定，最低分辨率1920×1080 |

### 1.5 演讲准备

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 主体内容可在10-15分钟内讲完 | 按照1页1-2分钟估算，超出则需精简内容 |
| 🟡 关键 | 每页备注栏写好口头补充要点 | 不要把备注写成演讲全文，写提示词即可 |
| 🟡 关键 | 已准备最常见的10个Q&A答案及对应附录页 | 提前预判：竞争对手、数据方法、执行风险、资源需求 |
| 🔵 加分 | 排练过至少1次完整演讲 | 找出卡顿点=找出逻辑漏洞 |
| 🔵 加分 | 已确认会场设备（分辨率、屏幕比例） | 4:3和16:9差异较大，提前适配 |

## 二、融资路演

### 2.1 叙事结构

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 开场60秒内建立问题感，让投资人有「痛感」 | 用一个真实用户故事或一组反直觉数据，而非公司简介 |
| 🔴 必须 | 包含标准9模块：Hook / Problem / Solution / Market / Traction / Model / Competition / Team / Ask | 每个模块可以是1-2页，不可缺失 |
| 🔴 必须 | 解决方案页有产品截图或Demo链接 | 「所见即所信」，纯文字描述说服力极弱 |
| 🟡 关键 | 整体叙事遵循「痛苦→希望→证明→邀请」四段式 | 先让人感受到问题有多真实，再展示你的答案有多可信 |
| 🟡 关键 | 结尾Ask页明确：融资金额、用途分配、18个月里程碑 | 「寻求A轮融资2000万，其中60%用于研发，40%用于销售，18个月内达到ARR 500万」 |
| 🔵 加分 | 设置「为什么是现在」页面，说明时机判断 | 市场成熟度、政策窗口、技术拐点，选最有力的一个 |

### 2.2 核心模块

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 市场页拆分TAM/SAM/SOM三层，并注明来源 | 只写TAM且无来源=最常见失分项；SOM要自下而上估算 |
| 🔴 必须 | 竞争分析页包含自身定位，不可只列竞品 | 用象限图标出自己的位置，说明差异化维度的选取依据 |
| 🔴 必须 | 团队页说明每人与业务的相关性，而非只列履历 | 「在X公司做过Y」不如「正因为做过Y，我们才能解决Z问题」 |
| 🟡 关键 | 商业模式页展示单位经济模型（LTV/CAC/毛利率） | 早期可以是假设值，但必须有测算逻辑 |
| 🟡 关键 | 牵引力页使用增长曲线，而非单点数据 | 「现有用户1000人」不如「3个月内从100增长到1000」 |
| 🔵 加分 | 附录备有：财务模型、技术架构、合规风险说明 | 投资人深度尽调时用，不主动讲但必须准备 |

### 2.3 数据与指标

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 所有数据有可溯源的出处 | 「市场规模2000亿」需标注：来源+年份+统计口径 |
| 🔴 必须 | 核心指标不超过3个，且逻辑自洽 | GMV、ARR、用户增速选最能代表业务健康度的 |
| 🟡 关键 | 不使用「亿级用户」「百亿市场」等无量化依据的描述 | 投资人会追问，无法回答比不说更损信任 |
| 🟡 关键 | 预测数据标注假设条件和测算方法 | 「若转化率达到5%，则ARR可达X」是可接受的表述方式 |
| 🔵 加分 | 展示核心指标的环比/同比增速，而非仅展示绝对值 | 增速=动能，是投资人最关心的维度之一 |

### 2.4 视觉风格

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 全篇风格统一，有品牌识别度（logo、色值、字体） | 路演PPT即品牌展示，设计感直接影响对团队能力的判断 |
| 🔴 必须 | 每页文字 ≤15字，主要靠口头表达 | 文字多=演讲者没准备好，投资人会低头读字而非看人 |
| 🟡 关键 | 关键数字用大字号（≥60pt）单独展示 | 「ARR 230%增速」应该是整页最大的视觉元素 |
| 🟡 关键 | 不使用模板化图表装饰（3D饼图、彩虹条形图） | 简洁的黑白折线图比花哨图表更显专业 |
| 🔵 加分 | 使用真实产品截图而非线框图 | 高保真截图传递产品成熟度信号 |
| 🔵 加分 | 团队照片使用同风格的专业照 | 混用不同背景、尺寸的照片会削弱整体感 |

### 2.5 演讲节奏

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 主体讲述控制在15-20分钟，严守时间 | 超时=对投资人时间不尊重，是负向信号 |
| 🔴 必须 | 备好10个常见Q&A及对应附录页 | 重点：竞争壁垒、市场规模方法论、退出策略、风险项 |
| 🟡 关键 | 能用一句话回答「你们是做什么的」 | 练习30秒电梯间表达，不能超过两个从句 |
| 🟡 关键 | 已针对不同类型投资人准备差异化版本 | 财务投资人关注回报，战略投资人关注协同，侧重不同 |
| 🔵 加分 | 提前研究目标投资机构的投资组合和偏好 | 能在介绍时提及「贵机构在X领域的布局」显示尽职调查能力 |

## 三、课程讲义

### 3.1 教学设计

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 封面页列出3条以内的可量化学习目标 | 用行为动词：「能够写出/分析/判断」，不用「了解/熟悉」 |
| 🔴 必须 | 开场有「为什么要学这个」的激活动机页 | 用真实场景或反直觉问题，而非直接进入概念讲解 |
| 🔴 必须 | 第2-3页呈现本课「知识地图」总览 | 帮学员建立认知框架，降低新知识的接收阻力 |
| 🟡 关键 | 结尾框架图与开头呼应，标注「已学」部分 | 闭环设计让学员感受到学习成就感 |
| 🟡 关键 | 每个知识点遵循：概念→原理→示例→练习四步 | 直接给定义再举例是错的顺序，应先给场景再提炼概念 |
| 🔵 加分 | 设置课前思考题，引导预激活 | 「你认为X的原因是什么？」先让学员有想法再给答案 |

### 3.2 内容组织

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 每页只讲一个知识点，不堆叠概念 | 两个并列概念须拆页，否则学员难以分配注意力 |
| 🔴 必须 | 抽象概念必须有具体例子对应 | 例子要贴近学员的真实工作/生活场景，不用通用教材式例子 |
| 🟡 关键 | 步骤型内容用编号，非步骤型用要点符号 | 顺序不可打乱的用1/2/3，可并列的用项目符号 |
| 🟡 关键 | 专业术语首次出现时给出定义和记忆钩子 | 「XX（英文缩写）即XXX，可理解为……」 |
| 🟡 关键 | 内容量与课程时长匹配（约1页/1-2分钟） | 45分钟课程约20-30页（不含封面、目录、互动页） |
| 🔵 加分 | 提供「知识精华卡」作为课后速查参考 | 1-2页浓缩核心公式/步骤/框架，可打印 |

### 3.3 互动设计

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 每15分钟设置至少1次互动或检查理解 | 互动形式：提问、小测验、讨论、练习题，任选其一 |
| 🟡 关键 | 互动页与内容页视觉风格有明显区分 | 换背景色或加边框，让学员「切换模式」 |
| 🟡 关键 | 测验题有正确答案说明，而非仅列选项 | 「答案是B，因为……」的解析比单纯公布答案价值大5倍 |
| 🟡 关键 | 案例分析题贴近目标学员的工作场景 | 不同行业/岗位的学员对案例的代入感差异很大 |
| 🔵 加分 | 设置「课堂挑战」：真实情境中的应用练习 | 比理论测验更能检验真实掌握程度 |

### 3.4 视觉表达

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 正文字号 ≥20pt，代码/公式使用等宽字体 | 等宽字体：Consolas / Source Code Pro / Courier New |
| 🔴 必须 | 图示与文字互补，不重复同一信息 | 文字说「流程有三步」，图示展示三步的关系，而非再写一遍 |
| 🟡 关键 | 不同功能的页面有视觉节奏变化 | 纯讲解页 / 互动页 / 总结页 / 案例页的视觉应有所区别 |
| 🟡 关键 | 复杂流程图优先使用分步骤页面，而非单页大图 | 单页大图让学员不知从哪里看起；分步骤逐一呈现更清晰 |
| 🔵 加分 | 关键公式/框架用高对比度配色突出显示 | 学员记笔记时，高亮内容就是「重点」信号 |
| 🔵 加分 | 插图/示意图使用一致的图形风格 | 混用照片+线稿+图标会让页面看起来像拼接 |

### 3.5 交付完整度

| 权重 | 规则 | 说明 |
|------|------|------|
| 🔴 必须 | 课后资料页包含：延伸阅读、作业说明、下节课预告 | 「课后资料」是学员最常复看的页面之一 |
| 🟡 关键 | 讲义文件有版本号和更新日期 | 格式：v1.2 \| 2024.10.15，方便学员辨认是否为最新版 |
| 🟡 关键 | 备注栏写有口头表达提示，而非照读的全文 | 关键词+过渡句即可，全文会让演讲变成「读稿」 |
| 🔵 加分 | 提供空白版讲义供学员填写笔记 | 空白版=复制一份，把答案/重点内容留白 |
| 🔵 加分 | 文件可在不同设备上正常显示（字体、排版不错乱） | 嵌入所有字体，避免依赖本地字体 |

## 附：评分权重说明

| 等级 | 权重 | 含义 |
|------|------|------|
| 🔴 必须满足 | 3分 | 缺失会显著影响PPT效果，上场前必须修复 |
| 🟡 关键项 | 2分 | 影响专业度和说服力，强烈建议满足 |
| 🔵 加分项 | 1分 | 提升质感和完整度，有余力时优化 |

**评分参考：**
- 90分以上：优秀，可以上场
- 75-89分：良好，建议修复关键项后上场
- 60-74分：及格，需要重点改进结构和内容
- 60分以下：待改进，建议重新梳理框架再制作
