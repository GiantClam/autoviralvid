# 数字人长音频分段生成 — Web 页面 E2E 测试用例

## 测试环境

| 项目 | 值 |
|------|------|
| 前端 URL | http://localhost:3000 |
| 后端 URL | http://localhost:8123 |
| 数字人图片 | `https://s.autoviralvid.com/test_dh_image_2026.png` |
| 测试音频 (152.6s) | `https://s.autoviralvid.com/test_dh_audio_10min_2026.mp3` |
| 声音模式 | 直接使用音频 |

---

## TC-WEB-001: 模板选择 — 数字人口播

**前置条件**: 打开 http://localhost:3000，已登录或 Dev Mode

**操作步骤**:
1. 在模板画廊中找到"品牌营销"分类
2. 点击"数字人口播"模板卡片

**预期结果**:
- [ ] 左侧表单显示"视频配置"面板
- [ ] 模板下拉框显示"数字人口播"
- [ ] 表单包含数字人特有字段：数字人形象图片、音频文件、声音模式、动作描述
- [ ] 音频文件字段下方显示提示"支持最长 30 分钟音频，长音频将自动分段生成并合成完整视频"
- [ ] "生成数字人视频"按钮初始为禁用状态（未填必填项）

---

## TC-WEB-002: 表单填写 — 完整参数

**前置条件**: TC-WEB-001 完成

**操作步骤**:
1. 主题输入框填写"数字人直播带货测试 - 长音频分段生成"
2. 数字人形象图片输入框填写 `https://s.autoviralvid.com/test_dh_image_2026.png`
3. 音频文件输入框填写 `https://s.autoviralvid.com/test_dh_audio_10min_2026.mp3`
4. 声音模式保持"直接使用音频"（默认）
5. 动作描述保持默认"模特正在做产品展示，进行电商直播带货"
6. 时长设为 600 秒
7. 方向保持"竖屏"

**预期结果**:
- [ ] 所有字段值正确填充
- [ ] "生成数字人视频"按钮变为可用（非禁用）
- [ ] 按钮文字为"生成数字人视频"

---

## TC-WEB-003: 提交生成 — 长音频自动分段

**前置条件**: TC-WEB-002 完成

**操作步骤**:
1. 点击"生成数字人视频"按钮

**预期结果**:
- [ ] 按钮立即显示加载状态（旋转图标 + "生成中…"）
- [ ] 页面进入 generating_videos 阶段
- [ ] 后端日志显示音频时长 ~152.6s，触发分段逻辑
- [ ] 后端日志显示分割为 3~4 段（每段 ≤ 45s）
- [ ] 后端日志显示每段任务依次提交到 RunningHub
- [ ] 右侧工作区从空闲状态切换到视频片段列表

---

## TC-WEB-004: 多任务进度展示

**前置条件**: TC-WEB-003 完成

**操作步骤**:
1. 观察右侧"视频片段"面板

**预期结果**:
- [ ] 顶部标题栏显示"视频片段"
- [ ] 进度条显示当前完成比例（如 "分段生成中 0/4"）
- [ ] 任务计数区域显示：总计 N、成功 0、处理中 N、等待 0
- [ ] 下方网格展示 N 个视频片段卡片（N = 分段数）
- [ ] 每个卡片显示 clip_idx、状态标签
- [ ] 卡片按 clip_idx 排序

## TC-WEB-005: 单个分段完成

**前置条件**: TC-WEB-004 完成，等待 RunningHub 处理

**操作步骤**:
1. 等待第一个分段完成（状态从 submitted → succeeded）

**预期结果**:
- [ ] 对应卡片状态变为绿色"成功"标签
- [ ] 进度条百分比增加
- [ ] 进度文字更新（如"分段生成中 1/4"）
- [ ] 完成的卡片显示"预览"按钮
- [ ] 成功计数增加

## TC-WEB-006: 所有分段完成 → 进入拼接阶段

**前置条件**: TC-WEB-005 中所有分段陆续完成

**操作步骤**:
1. 等待所有分段任务都变为 succeeded
2. 观察页面状态变化

**预期结果**:
- [ ] 进度条 100%
- [ ] 状态文字变为"正在合成完整视频..."（琥珀色，带旋转图标）
- [ ] 底部操作栏显示"所有分段已完成，正在自动合成完整视频…"
- [ ] **不显示**手动"合成视频"按钮（因为自动拼接进行中）
- [ ] 后端日志显示 `Starting auto-stitch for run_id=...`

---

## TC-WEB-007: 自动拼接完成 → 最终视频

**前置条件**: TC-WEB-006 拼接完成

**操作步骤**:
1. 等待拼接完成（后端日志显示 `Auto-stitch completed`）

**预期结果**:
- [ ] 页面自动进入 completed 阶段
- [ ] 顶部状态显示绿色"全部完成"
- [ ] 底部操作栏出现"下载"按钮
- [ ] 下载按钮链接指向最终拼接视频 URL（包含 `_dh_final.mp4`）

- [ ] 项目列表中该项目状态更新
